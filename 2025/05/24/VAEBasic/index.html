VAEBasic

变分自编码器

基于 MLP，我们可以轻松完成手写数字分类的任务。这固然是令人欣喜的结果，但是我们对神经网络的开发不能止步于分类——毕竟，会分类只是懂得学习，能生成才逐渐接近创造。

在这篇文章里，我们将研究数字生成任务（这里只要求能生成出数字，不要求生成指定的数字）。第一部分将从直觉出发，介绍简单的自编码器，并探讨其缺陷；第二部分将从通过对编码空间的可视化观察，提出改进方案——变分自编码器；第三部分将详细解释 VAE 背后的原理；第四部分本想讲解代码的具体实现，但鉴于之前已经有比较详细的解读，这里仅着眼于之前没有提及的一个实现细节——重参数技巧。

1. 简单的自编码器

1.1 一个自然的想法

观察我们用于手写数字分类的 MLP 的结构，我们发现它由三部分构成：输入层、隐藏层、输出层。根据之前的学习，它的作用是提取特征，所以可以拆分成两半理解它的功能： - 输入层到中间某个隐藏层：提取特征，将输入向量映射至这个隐藏层代表的“隐空间”中； - 这个隐藏层到输出层：根据特征提供的信息，得到它是各个数字的概率。

基于这个观察，我们有一个很自然的想法：只保留前一半的神经网络，再训练一个将隐空间向量还原为数字图像的神经网络，这样就能生成了。

这其实就是 简单的自编码器。左半边的网络是一个将输入图像向量编码到隐空间向量 z 的神经网络，称为编码器；右半边是一个将隐空间向量 z 重构为一个图像向量的神经网络，称为解码器。

这里要注意一点：我们其实并不知道隐空间中的向量 z 是否真的可以代表特征，但是我们相信它可以。

训练的时候，我们希望对于每张手写数字图片，重构出来的图片和原图片尽可能相等。训练好之后，我们在低维的隐空间任意采样一个 z，就可以通过解码器进行生成了。

然而，想法很好，模型的实际生成效果却不尽如人意。

1.2 问题出在哪

假如每个样本都重构得很好，那么我们可以将隐空间向量 z 当成是输入向量 x 的等价表示。现在我们来思考一个问题：这些 z 覆盖的空间长什么样子？

为什么我们要关心这个问题？因为编码方式是我们指定的，编码结果越规整、连续，我们后续采样的时候也就更容易得到好的结果，也就是模型的泛化能力较强。所以我们希望我们的编码方式可以产生比较理想的编码向量分布。例如下面这四个例子中，最后一个编码空间的分布是最理想的：

2. 变分自编码器

为了达到更好的生成效果，我们关心编码空间的如下 4 个问题： - 1.所有编码向量覆盖一个什么样的区域？ - 2.是否有未知的真是样本对应空白之处的向量？ - 3.有没有“孤悬海外”的向量？ - 4.有没有办法让编码空间更加规整？

简单的自编码器没有特别的约束，因此很难回答上面的问题。变分自编码器应运而生。从编码空间的角度来看，它希望做到两件事： - 1.让编码空间更加规整 - 2.让编码向量更加紧凑。

为了实现这样的目标，变分自编码器引入了后验分布 p(z|x)。关于为什么是后验分布，我们将在第三部分探讨，现在，我们从可视化的几何视角来直观感受一下后验分布的作用。

2.1 由点到面

直观来看，我们可以将后验分布理解为一个“椭圆”。原来的每个样本对应编码空间中的一个编码向量，也就是编码空间中的一个点，再多的“点”也没有办法把一个“面”覆盖完；引入后验分布之后，每一个将会对应一个“椭圆”，用“面”来覆盖“面”，就容易得多了。

为什么是椭圆呢？因为高斯分布是比较容易处理的一类概率分布，在后续计算 KL 散度的时候也会较为方便，所以在变分自编码器中我们令后验分布为高斯分布。而在概率语言上，椭圆对应着“假设 p(z|x) 各分量独立的高斯分布”，所以我们使用高斯分布，也就对应着椭圆。使用其他分布那就对应其它的形状，例如矩形对应均匀分布。

现在每个样本 x 都对应一个椭圆，而确定一个椭圆需要两个信息：椭圆中心、椭圆轴长，它们各自构成一个向量，并且这个向量依赖于样本 x，记为 μ(x), σ(x)。既然整个椭圆都对应着样本 x，我们便要求椭圆内任意一点都可以重构 x，所以训练目标为：

2.2 空间正则

“椭圆”可以“让编码向量更加紧凑”，但是不能实现“让编码空间更加规整”。现在我们希望编码向量满足标准正态分布（直观上，即所有的椭圆覆盖的空间组成一个单位圆）。这也就是“正则”的含义，可以理解为对分布形态的一种约束。

3. 背后的原理

3.1 一种错误的理解

考虑这样一个情景：现在我们有一批数据样本 {X1, X2, …Xn}，其反映的真实集合为 X。我们本想根据 {X1, X2, …, Xn} 直接得到 X 的分布 p(X)，如果可以的话，我们直接根据 p(X) 采样，我们就可以得到所有的 X 了。然而，这样的理想很难实现，于是我们引入一个中间层 Z，将分布改写为： p(X) = ∑Zp(X|Z)p(Z) 这里混用求和和求积分。此时 p(X|Z) 描述了一个由 Z 来生成 X 的模型。现在我们提出一个先验假设：Z 服从标准正态分布，即 p(Z) = 𝒩(0, I)。如果这个假设成立，那么我们直接在标准正态分布中采样一个 Z，之后直接根据 Z 来算 X，轻松实现生成。接下来要做的就是结合自编码器实现重构，保证训练集提供的有效信息没有丢失，再叽里呱啦一通推导即可实现模型。模型框架如下：

3.2 反着来看看

现在，我们不要把真实样本当成“输入集”，而将其视为“输出集”。

什么意思呢？我们现在想要学习的其实是：真实样本是从什么样的编码空间被如何还原的。前半段对应编码器的目标，后半段对应解码器的目标。基于这样的认识，我们认为这些真实样本都是从某个编码空间被解码出来的，因此样本实际上是结果，而编码器要做的是根据结果反推原先编码空间的分布，因此我们要进行 后验分布 假设！

同时，为了实现采样结果和样本的一一对应，从而得以科学地计算损失函数，我们还需要为每个样本提供一个专属于它的后验分布。

形式化地说，给定一个真实样本 Xk，我们假设编码空间中存在一个专属于 Xk 的分布 p(Z|Xk)，并进一步假设这个分布是（独立的、多元的）正态分布。后续我们将训练一个生成器 X = g(Z)，希望能够把从分布 p(Z|Xk) 中采样出来的一个 Zk 还原为 Xk。现在 p(Z|Xk) 专属于 Xk，所以我们有理由说从这个分布采样出来的 Z 应该还原到 Xk 中。

问题又出现了：每一个 Xk 有自己的专属正态分布，其具有均值 μ 和方差 σ2，那我怎么设计这样的参数呢？

在神经网络的时代，这个问题非常简单：我们使用神经网络进行拟合。具体来说，我们构建两个神经网络 μk = f1(Xk), log σ(k)2 = f2(Xk) 来计算。这里拟合 log σ(k)2 而不是直接拟合 σ(k)2 是因为后者总是非负的，而前者可正可负（这一点我还没有完全理解）。

到这里，我可以知道专属于 Xk 的正态分布，然后从专属分布中采样一个 Zk 出来，经过一个生成器 ，之后放心地最小化诸如 的损失函数，毕竟 Zk 从 Xk 的专属分布中采样出来，生成器自然应该要把开始的 Xk 还原出来。 此时 VAE 的示意图如下：

3.3 最后的拼图

让我们看看完全按照上述流程训练会得到什么结果。

首先，我们希望重构出来 X，也就是最小化损失函数。但是这个过程受到噪声的影响，因为 Zk 是通过重新采样得来的，不是 encoder 直接计算出来的。幸好这个噪声是我们人为引入的高斯噪声，是通过神经网络算出来的。所以模型为了重构得更好，肯定想法设法让噪声减小，也就是方差为 0。方差为 0 那就没有随机性了，所以模型其实在 慢慢退化为简单的自编码器。

天哪！这看上去太可怕了！我们要怎么应对这种情况呢？

想法其实也非常简单，它退化是因为退化可以让损失函数减小而没有别的惩罚 ，那我们只需要在损失函数里为退化这一过程设计惩罚机制，就可以防止噪声为 0 了。

具体来说，VAE 让所有的 p(Z|X) 向标准正态分布看齐。因此，原论文直接算了该分布和标准正态分布的 KL 散度 KL(𝒩(μ, σ2)‖𝒩(0, I)) 作为额外的 Loss，也就是刚刚第二部分出现的那个式子： 这里的 d 是隐空间 Z 的维度。

这样的操作是保证了模型的生成能力的。为什么呢？因为如果所有的 P(Z|X) 都很接近标准的正态分布 𝒩(0, I)，那么根据定义有 p(Z) = ∑Xp(Z|X)p(X) = ∑X𝒩(0, I)p(X) = 𝒩(0, I) 从而达到我们的先验假设：p(Z) 是标准正态分布。之后我们放心地从 𝒩(0, I) 中采样来生成图像即可。虽然这里我们也得到了和错误的想法中相同的先验假设，但思考角度的不同使得其实际含义是完全不一样的。

3.4 小小的总结

VAE 本质上在我们常规的自编码器的基础上，对 encoder 的结果（在VAE中对应着计算均值的网络）加上了“高斯噪声”，使得结果 decoder 能够对噪声有鲁棒性；而那个额外的 KL loss（目的是让均值为0，方差为1），事实上就是相当于对 encoder 的一个正则项，希望 encoder 出来的东西有零均值。

另外一个 encoder（对应着计算方差的网络）是用来动态调节噪声的强度的。直觉上来想，当 decoder 还没有训练好时（重构误差远大于 KL loss），就会适当降低噪声（KL loss 增加），使得拟合起来容易一些（重构误差开始下降）；反之，如果 decoder 训练得还不错时（重构误差小于 KL loss），这时候噪声就会增加（KL loss 减少），使得拟合更加困难了（重构误差又开始增加），这时候 decoder 就要想办法提高它的生成能力了。

不过这里存在一个矛盾：如果 p(Z|X) 完全精确等于标准正态分布，那么 p(Z|X) 其实与 X 无关，从而重构效果将会极差，正如我们之前错误的理解那样；而 p(Z|X) 不是标准正态分布，那我们的采样策略就存在问题，生成能力就会打折扣。VAE 的损失函数设计其实做了一个折中：“对抗式的”损失函数调整使得 p(Z|X) 保留了一定的 X 的信息，同时其近似等于标准正态分布，从而既有比较好的重构效果，也保留了生成能力。

4 实现

这里我们只关注一个点：重参数技巧（reparameterization trick）。

现在我们要从 p(Z|Xk) 中采样一个 Zk 出来，尽管根据对后验分布的假设我们知道 p(Z|Xk) 是正态分布，但是均值方差毕竟要通过模型进行拟合，所以我们需要通过采样提供的信息来优化均值方差的模型。但是“采样”是一个离散的过程，不可导。因此我们利用

来说明 (z − μ)/σ = ϵ 是服从均值为 0，方差为 1 的标准正态分布的。这个时候我们得到： > 从 𝒩(μ, σ2) 采样一个 Z，相当于从 𝒩(0, I) 中采样一个 ϵ，然后让 Z = μ + ϵ × σ。

于是我们将从 𝒩(μ, σ2) 采样变成了先从 𝒩(0, I) 中采样，再通过参数变换得到从 𝒩(μ, σ2) 中采样的结果。这样一来，“采样”这个离散的操作被固定下来不参与梯度下降，改为采样过程的变换，即获得采样结果的过程参与，这就使得整个模型可以训练了。
