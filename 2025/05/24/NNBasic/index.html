神经网络基础

神经网络是一种相当优美的程序设计范式。与传统的处理策略——将要解决的问题精确拆分和设计成计算机能够高效解决的子任务——不同，在使用神经网络的时候，我们不会直接告诉计算机我们应该怎么解决这个问题。相反，它会自己从数据中学习，找到相应的解决方案。

在这篇文档中，第一部分将会基于 MLP 介绍神经网络的基本构成，以及简洁的数学描述；在第二部分会介绍梯度下降方法，为神经网络的调整方向提供一种思路；第三部分会简要介绍矩阵求导的知识以及 numpy 的广播机制，为数学表述和代码实现的简化提供依据；第四部分会进一步阐释反向传播，它为模型自我调整提供具体实现路径。

1.2 多层感知机

有了 sigmoid 感知器，我们就可以搭建最基础的神经网络——多层感知机（multilayer perceptrons，或 MLPs）。

MLP 由若干层全连接层组成，即每一层的每个节点都和上一级的所有节点有连边。其中第一层为输入层，中间经过若干层隐藏层，最后从输出层中获得结果。值得一提的是，隐藏层的“隐藏”二字背后并没有深刻的含义，仅仅是因为它不是输入层也不是输出层。

在第 i 个隐藏层的第 j 个节点，其输出设为 yi, j ，它和上一层的各个节点都有连边。不妨设上一层节点的数量为 ni − 1，这一层为 ni。上一层节点的输出为这一层节点的输入，例如从上一层第 k 个节点到这一层第 j 个节点的输入记为 yi − 1, k = xk，相应边上权值记为 wk, j，则 yi, j 可表示为

考虑这样一个一般性的问题：假设 C 是有 m 个变量 v1, v2, …, vm 的单值函数。自变量的变化量可以表示为函数 Δv = [Δv1, Δv2, …, Δvm]T。在 ‖Δv‖ 足够小的时候，我们可以用微分来近似替代变化量，从而 根据数学上的定义，梯度向量为该点处函数对各变量偏导构成的向量，即 从而变化量可以表示为 ΔC = ∇C ⋅ Δv

我们回到使用梯度下降方法的最初目的来思考。我们想要做的无非是更改参数从而实现两个目的： - 使得损失函数的值下降，从而提高拟合效果 - 值下降的速度尽可能快，从而提高训练效率

根据这两个目的，按照梯度下降的策略就非常显然。 - 一方面，一个非 0 数的平方的相反数肯定是一个负数，因此取 Δv = −η∇C 时，其中 η > 0，则 ΔC = −η(∇C)2 在未达到极值点的时候肯定是小于 0 的，一定会下降； - 另一方面，为了衡量下降快慢，我们考虑固定 Δv 的长度，研究什么时候 ΔC 值最小。显然 Δv 与梯度向量方向相反的时候值最小。

综合以上的观察，我们说梯度的反方向是下降最快的方向。

3 矩阵求导与广播机制

3.1 矩阵求导

正如第一部分中我们看到的那样，在神经网络中，相邻的两层间信息的传递可以用矩阵来非常简洁地表达： yi = σ(Wixi + bi) 这似乎启示我们可以直接将矩阵当作普通的数字来进行操作。

事实上，这样做确实是合理的，因为我们使用矩阵的初衷不过是把具有相同结构的式子提炼一下，使用简单的符号进行统一表达。对于加减乘除（这里的除理解为和加减类似的运算逻辑：两个大小相同的矩阵，相应位置上的元素相除；后续我们在进行除法的时候使用的也是这种除法），理解起来是比较容易的。但是为了在梯度下降等方法中使用这种记号，我们要对矩阵求导进行一些定义。

和我们刚刚使用矩阵描述层间运算的出发点一样，矩阵求导的本质只是把标量求导的结果排列起来，至于按行排列还是按列排列其实都没问题。但是行向量列向量随便写，那么结果就不唯一。因此为了解决这个问题，我们引入求导布局。

3.2 广播机制

在进行数学上的分析的时候，我们可以将矩阵按照数字一样进行操作。现在我们希望在写程序的时候也可以直接把矩阵当成普通的变量来计算。

python 库 numpy 的广播机制正是这样一种机制。它是 numpy 对不同形状的数组进行数值计算的方式。如果两个数组形状相同，则进行算术运算的时候是直接对应位上的元素进行算术运算，这和我们的需求是相同的。所以在实际程序中我们也无需考虑数据类型，直接将矩阵当作普通变量即可。
